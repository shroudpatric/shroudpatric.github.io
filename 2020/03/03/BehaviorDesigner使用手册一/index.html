Behavior Designer 的四个基本类型 Task

Action

Action tasks are probably the easiest to understand in that they alter the state of the game in some way。
Action 类型的 Task 或许是最容易去理解的因为他表示了游戏的一些状态。
你可以把 Action 理解为一个动作，一个方法。

Conditional

Conditional tasks test some property of the game.
Conditional 类型的 Task 用来测试游戏的一些属性。
你可以把 Conditional 理解为一个判断语句，类似于 If 或者 Switch 等等。

Composites

The decorator task is a parent task that can only have one child.
Decorator 类型的 Task 是只有一个子节点的父节点。
该类型的节点常用作打断节点来使用，后面会举例说明该节点的作用。

Decorators

Composite tasks are a parent task that hold a list of child tasks.
Composite 类型的 Task 是一系列子节点的父节点，比如说行为树中有一个 Task 叫做 Sequence，就是一个 Composite 类型的节点，这个节点就用来表示“和”，它会依次执行该子节点的每个节点，不过一旦有一个节点返回失败，那么整个 Sequence 节点也就返回失败了。
