EZReplayManager使用说明

Created2020-03-09|Updated2021-04-17
|Post View:

EZReplayManager 使用记录

最近增加的回放需求，首选是 EZReplayManager,不过最后应该不会使用这个插件，因为实在是最后生成的本地文件太大了，而且这个本地文件还跟你的回放场景有关，也就是说如果你所需要回放的是一个偏庞大或者偏细致的场景，那么最后生成的本地文
件都会很大，所以大家看需自行考虑吧。

插件的导入

首先声明，笔者所使用的插件版本是 1.5 版本，在导入过程中，尝试了 Unity 2018 和 2019 版本都不行，原因与2017 后粒子系统的大改有关，而且在该插件中又对粒子系统也进行了回放处理，所以在 2017 以上的版本中都无法使用。

我在导入插件之后报了如下错误：

这个工程我是在 Unity 2017 的 LTS 版本里面导入的，删除了两个 JavaScript 脚本就解决了。

这个错误是我在运行插件里面的案例2出来的，错误是动画丢失，在场景里面找到角色然后找到他的动画机，通过动画可以找到模型，然后将丢失的动画拖入动画机里面就好了。

这个界面也很好去理解，播放，暂停，倍速。

如果你选择了保存回放，那么首先，在 PC 平台下,你的回放会保存为一个名为 “example2.ezr” 的文件，这个名字是官方写好的，不过后期可以修改文件的名字但是不能修改后缀名，保存路径是在你的总项目下面，和Assets 平级的路径下面。

案例原理解析和 API 记录

官方案例二的场景中有一个预制体，如下图：

该预制体有三个属性比较重要在这里先说明一下：

  • Recording Interval : 默认是 0.05，意味着每秒记录 20 帧，所以一般你要根据你游戏的实际情况选择录制间隔的参数，这里建议每次调整的幅度为 0.01，当参数为 0.04 的时候 游戏录制就是 25 帧。

  • Game Objects To Record : 这个参数是一个游戏列表，你需要将你所有记录的游戏体都放到这个列表中，你也可以用下面的 API 来进行动态操作。

括号中的参数就是你需要记录的动态添加的物体。

  • Components And Scripts To Keep At Replay ：在回放过程中也不希望停止运行的脚本或者组件，注意这是一个string 类型的数组。该方法用的是循环寻找判定的，会比较消耗性能。

下面是一些比较常用的 API ：

  • EZReplayManager.record(); //开始录制
  • EZReplayManager.stop(); //停止录制
  • EZReplayManager.play(0); //播放的速度 参数范围是 1-5 小于 0 减速
  • EZReplayManager.saveToFile(filename); //将录制的文件保存为 filename，这里这个参数的名字可以是单纯的参数名字，那么这里就会直接保存到和 Asset 同级目录下，注意不是保存在 Asset 目录下，也可以是包含绝对路径的参数名字。

EZReplayManager 重点

默认情况下，这个插件记录的只是放到纪录列表里面游戏物体的坐标，旋转，和游戏物体是否发射了粒子系统(所以在粒子系统大改的2018以上版本中会有一个报错)。在第一次开始录制的时候，他会检查需要录制的游戏体列表也就是这个列表
：”gameObjectsToRecord”.将这个列表中所有的游戏体都做成预设体保存下来，这也就是为什么场景越复杂越细致最后得到的回放文件越大的原因。如果你想要回放，那么你需要先加载录制过程中所有形成的预设体。
下面这个 API 可以将你想录制的游戏体做成的预设放到特定的位置保存。
