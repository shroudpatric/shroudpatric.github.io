点图 Point Graph

基本点图point graph 是最基本的一种图结构，该图由一系列在空间中相互连接被称作 nodes 或者 waypoints的点构成。
point graph 需要一个物体的变换组件作为根节点，该物体的子物体会被搜索，每一个子物体都会被当做一个节点，如果 recursive（遍历）选项被启用的话，那么所有子物体的子物体也会被遍历搜索。图会检查在任何节点之间是否有链接，首先它会检查节点之间的距离是否太大（maxDistance）然后它会检查轴对齐距离是否太高。轴对齐距离，命名为 limits，是非常有用的，因为 AI 不能爬非常高的坡，而链接的节点之间又非常高，如果在相同的Y轴就应该可以寻路。limits 和 maxDistance 在被设置为0的情况下等同于0。最后会在节点之间用 raycasting（可被选择性的加厚） 检查是否存在障碍物。
使用 raycasting 时需要注意一件事，场景中节点应该放在地面上方一小段距离或者让地面的层不在 raycast 的检测中，来避免 raycast 检测的时候投射到地面上。在搜索节点的时候也可以用标签来选择。对于比较大的图来说，用默认的设置扫描成图可能会消耗一点时间。如果你有 pro 版本你可以开启 optimeziForSparseGraph 功能在大多数情况下都可以很大程度上减少配置的时间。注意：不支持 linecast 因为各种原因。

使用说明

  • 点图使用的时候直接将组成点网的物体拖入 Astar Path 组件中图的 Root 即可。
  • 注意最大距离，点图不适用于密集图，最大距离处于不合适的范围时会导致寻路出现非常怪异的现象。
