RVO 局部回避和 环境回避

 在AStar中有一个概念叫做局部回避，也就是RVO（Reciprocal Velocity Obstacles），使用RVO可以让你的友军避免在同一个图（Graph）产生大范围挤压碰撞，通俗的讲就是在有多个友军在寻路返回相同的结点时产生回避现象。
 首先说明，RVO是和碰撞体不兼容的，据大佬说RVO允许在小范围内挤压或者说重合，但是碰撞体是不允许挤压或者重合的（非触发），而目前在我的实验中，所有的寻路单位都是带有RVO的，所以如果不对RVO和碰撞体进行处理会产生逻辑相悖的情况，即碰撞体相撞产生了力但是RVO允许小部分挤压重合继续碰撞，使得最后的情况非常糟糕。
 我的实验是需要寻路的两个物体产生碰撞效果的，所以调整了RVO和碰撞体基本大小相同或者RVO略小于实际物体，不过又引来了新的问题，环境的边界我是用的碰撞体进行限定的，但是寻路的时候Agent不会根据Obstacle层来回避边界，于是当Agent在边界然后和边界的碰撞体产生碰撞就会造成人物的抖动。
 一开始为了解决这个问题我尝试将 AStar Path 组件中的 Diameter 调高，这样虽然会使得边界部分的不可寻路区域变宽，但是当Agent的碰撞体大小超过了这个宽度的时候，这个解决办法就没用了，而且这种方法还会带来另一种影响就是整个 Graph 的路线都会被改动，有点舍本逐末的意思。
 后面我发现了另一种方法，Local Avoidance。在 AStar 组件中有一个脚本叫做 RVO Square obstacle，奇怪的是我直接在添加组件的地方输入RVO不会有这个组件，但是输入 Square 出现的组件前面又带有 RVO 三个字母，不是很懂这个地方，这个组件里面可以选择障碍类型，层级，高度，大小，中心位置。按照我的需求，Obstacle Mode 我选择了 Keep out，Layer 选择了和 Astar Path 组件中 Collision testing 里面 Obstacle Layer Mask 相同的层。


 最后的效果就是当 Agent 寻路到了这附近的时候会自动避开这个环境，不过需要注意的是，这个障碍物在每次生成 Graph 的时候都会重新计算一遍，所以如果是动态生成图比较频繁的情况下需要考虑性能消耗。
