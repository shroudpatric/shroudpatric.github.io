Grid Graph 的简单总结和翻译

Grid Graph 所做的事情就像他的名字一样，生成网格状的节点。Grid 类的图可以很好与你切合如果你的世界是基于网格状建立的。
注意以下几点：
你可以在运行的时候更新图，比如塔防和RTS类游戏。
在任何场景中使用它，你都可以得到一个不错的图的结果并且使用最小的配置量。
支持 raycast 和 funnel 算法。
可预见的模式。
可以添加惩罚与可行走的值从供给的图像上。
完美的适合地形世界因为他可以让斜坡来决定区域的可通性。


The Snap Size

The Snap Size 这个按钮会将图内部的尺寸明确分割为包含当前确切数量的若干节点(nodes)。换而言之不是100.3个节点而是确切的100个节点。这样会使得“中心点”与图更准确的切合。（不过笔者暂时还没有找到）

在运行时更新

任何实现了 IUpdateableGraph 接口的图都可以在运行时更新。对于 grid graph来说，这是一个非常好的功能你只需要更新一小部分的网格不会造成任何（看起来像重新扫描造成的）停滞。
例如你想在场景中生成(Instantiate)一个圆形的障碍物并且你想要更新你生成圆形障碍物处的网格，你就可以这样做：
AstarPath.active.UpdateGraphs (ob.collider.bounds);
在这里 ob 表示你刚刚生成的障碍物物体。
正如你所看见的，UpdateGraphs 方法传入了一个 Bounds 类型的参数并且该方法会通知所有的可更新的图。
grid graph 会更新那一片区域和环绕着该区域的一小部分，这一小部分正好等于 collision testing 中 diameter 参数的一半。
可以查看图的运行时更新来获取更多关于图类型的运行时更新信息。

Hexagon graphs

图可以被配置为六边形图来工作通过一些简单的设置。在4.1.x版本之后 grid graph 有一个 “Shape”的下拉列表。如果你将它设置为六边形那么该图就会变成六边形图。通常你可能会想要旋转图正负45°。
请注意虽然 snapping 到最近的节点不是像你所期盼六边形图那样，但是这也最够接近（现实）甚至你也可能不会注意到差别。

Tree colliders

似乎 Unity 只会在游戏运行之后才会激活 tree colliders。由于这个原因，grid graph 不会在运行模式之外选定 tree colliders 但是会在开始的时候选定他们。如果在这种情况下仍然没有选定哪些 tree colliders 确保 tree 的身上真的附着了碰撞体并且 tree 的预设体处于正确的层级（该层级应该包含在 Collision Testing 模块中）。

案例实践部分

案例构建

首先建立一个如笔者这样的场景：

  • 在场景中新建一个 Plane，层级和标签都保持不变。
  • 建立一个空物体，重命名为 A* ，然后添加脚本 Astar Path,如果不能在下面直接添加就找到插件的 Core 文件夹，这个脚本就在这里，拖到游戏物体上。
  • 然后仔细观察这个组件，选择 Add New Graph,然后选择 Grid Graph.
  • 这个时候选择快捷键 R,切换到游戏物体的缩放界面，将这个 Graph 的大小缩放至和我们的 Plane 一样大，然后查看 Astar Path 组件，最下面有一个 Scan 按钮，点击之后就生成了接下来的寻路区域。
  • 在场景中添加障碍物，如图。
  • 将这些障碍物放到一个物体下面，然后添加一个 Layer,一般命名为 Obstacle,然后将刚才添加的障碍物都切换为这个层。
  • 如果这个时候你点击刚才的 Scan 按钮，你会发现障碍物变成了如下这个样子，这是因为 Astar 目前没有将这些障碍物识别为障碍物，所以回到刚才的 Astar 组件界面，在 Collision testing 下面的 Obstacle Layer Mask 中选择刚才你新建的障碍物层，顺便修改一下 Diameter 的参数，这里这个参数的意思就是障碍物旁边的禁止访问缓冲区距离，一般来说，这个参数要配合着场景中寻路的 Agent 一起调整，因为 Agent 在寻路的时候并不会考虑其他的，还是以一个点来寻路，但是正常情况下 Agent 都是有模型的，所以这个模型的半径一定要小于这个缓冲区的距离，不然就会出现模型和禁止寻路区域的边界发生穿模或者挤压现象。

  • 这里做一点拓展，如果寻路的 Agent 的模型大小不一样，或者 Agents 之间的模型差距较大怎么办，那么就需要将这些 Agents 进行分类，然后建立多个 Graph 分别寻路。
  • 下一步是在场景中放置一个 Agent 和一个 寻路 Target
  • Agent 是由一个空物体下面挂载一个 Capsule 构成，Target 由一个空物体下面挂载一个 Cube 构成。
  • 如果这个时候你选择 Astar 组件的 Scan 按钮你会发现 Agent 和 Target 都被扫描进了环境中，所以你需要将寻路和被寻路的物体从环境中分离出来，这时候新建 Player 层，然后修改 Agent 和 Target 的层级为 Player，接着修改 Astar 组件的 Height testing 中 Mask，将 Player 层级在这里取消选中。
  • 然后在 Agent 上面挂载三个脚本，分别是 Seeker， AI Path， AI Destination Setter， 然后将 Target 拖进 AI Destination Setter 的 Target 中。
  • 最后点击开始就可以看到寻路效果了。
